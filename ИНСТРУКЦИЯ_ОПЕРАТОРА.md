# АРМ оператора — инструкция и чек-лист тестирования

## Запуск

1. Дважды щёлкнуть `АРМ.bat` в папке программы.
2. В строке состояния внизу должно быть: «Сканер: подключён ✓».

Обновление программы: в папке программы открыть командную строку → `git pull` → перезапустить АРМ.

## Основной сценарий (клиент пришёл)

1. **Считать паспорт** — положить паспорт на Regula разворотом вниз и нажать
   «📷 Считать паспорт (Regula)». Поля (ФИО, номер, даты…) заполнятся сами.
   - Быстрый вариант: «📷📄 Считать паспорт + Согласие» — сразу сформирует
     согласие и откроет его для печати.
   - Ещё быстрее: включить галочку «Автозахват» — тогда паспорт считывается
     сам, как только его положили на сканер (без нажатий).
2. **Проверить поля** — при необходимости поправить вручную
   (кнопка «🔤 Транслитерация lat→cyr» переводит латиницу в кириллицу).
3. **Отметить услуги** галочками (договор на обучение → акт добавится сам).
4. **«📄 Сформировать комплект»** — создаст согласие + договор + акт в папке
   клиента, запишет клиента в Excel-реестр, откроет папку.
5. **Страницы паспорта**: «📚 Сканировать страницы (Regula)» — класть развороты
   по очереди, после каждого отвечать «Да»/«Нет».
6. **«🗂️ Сформировать развороты 2×2»** — соберёт из страниц PDF
   (А4 книжный, 4 разворота на лист).
7. **Сшитые документы** (переводы, медкомиссия): «📑 Сканировать документ (Kodak)».
8. Печать: «🖨️ Печать последнего PDF» или открыть документ и распечатать.

## Что где лежит

- Папки клиентов: сетевой архив (путь виден в строке состояния, «Archive: …»).
- Шаблоны документов (`согласие.docx`, `договор_обучение.docx`, `акт.docx`):
  папка из `arm_config.json`, ключ `templates_dir`.
- Настройки: `arm_config.json` рядом с программой.

## Если что-то не работает — что прислать разработчику

Любая ошибка показывается окном с текстом. **Скопируйте текст целиком** (или
сфотографируйте) и отправьте — особенно строки «Диагностика: {...}».

| Симптом | Что проверить / прислать |
|---|---|
| «Сканер: не найден ✗» | Подключён ли Regula по USB; открывается ли фирменная программа Regula |
| «документ не обработан за N c» | Паспорт лежал ДО нажатия? Прислать текст с «Диагностика» |
| «NAPS2: страниц не получено» | Установлен ли NAPS2; есть ли профиль `kodak` (NAPS2 → Профили) |
| «Не найдены шаблоны» | Лежат ли `.docx`-шаблоны в папке шаблонов, точные имена |
| Вместо страницы паспорта только фото | Прислать текст «Диагностика снимков» из окна после считывания |
| Данные считались с ошибками | Прислать строку «MRZ, как прочитано: [...]» из предупреждения |

## Чек-лист тестирования (пройти по порядку)

- [ ] Запуск АРМ, статус «Сканер: подключён ✓»
- [ ] Считывание внутреннего паспорта РФ: ФИО с отчеством, номер (10 цифр),
      дата рождения, дата выдачи — заполнились верно
- [ ] Считывание иностранного паспорта: ФИО, номер, срок действия
- [ ] Транслитерация: имена с «й» (Сергей, Николай) переводятся правильно
- [ ] Автозахват: включить галочку, положить паспорт — поля заполнились сами
- [ ] «Считать паспорт + Согласие»: согласие открылось, данные подставлены
- [ ] Скан страниц: 3–4 разворота подряд, без лишних окон между страницами
- [ ] Развороты 2×2: PDF книжный, страницы повёрнуты правильно, видна ВСЯ
      страница (не только фото) — если фото, прислать диагностику
- [ ] Сформировать комплект с договором: созданы согласие + договор + акт,
      клиент попал в Excel-реестр «обучение»
- [ ] Kodak: сшитый документ сканируется в PDF
- [ ] «Открыть папку клиента» — папка открывается
- [ ] Печать PDF
